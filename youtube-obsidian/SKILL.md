# youtube-obsidian — 유튜브 → 분류 → Obsidian 저장

채팅에 붙여넣은 유튜브 링크를 **자막+메타데이터로 분석 → 카테고리 분류 → 볼트의 맞는 폴더에 노트 저장**한다.
링크 여러 개면 각각 반복한다.

## 사전 준비 (PC마다 1회)

1. **파이썬 패키지 설치** — 추출에 필요:
   ```bash
   python -m pip install youtube-transcript-api yt-dlp
   ```
2. **볼트 루트 설정** — 볼트 경로는 **PC마다 다르므로 하드코딩하지 않는다.**
   `<skillDir>/vault_root.txt`에 볼트 루트 한 줄을 적어둔다(예: `C:\Users\yejun\Yejun`).
   - 이 파일이 없으면 사용자에게 볼트 루트를 물어 **그 파일에 저장**한 뒤 진행한다.
   - 카테고리 폴더는 이 루트 기준 **상대경로**(아래 표)라 루트만 PC별로 바꾸면 된다.

## 워크플로우

### 1. 링크에서 재료 추출 (스크립트)

```bash
python "<skillDir>/scripts/extract.py" "<유튜브 URL 또는 ID>" "<출력.json>"
```
- 출력 json(UTF-8)을 **Read**로 읽는다. 담기는 것: `video_id, url, transcript, transcript_ok, transcript_lang, title, channel, upload_date(YYYYMMDD), description, hashtags, duration`.
- **인코딩 주의**: 한글은 이 스크립트(=youtube-transcript-api + yt-dlp 라이브러리)를 통해야 안 깨진다.
  yt-dlp를 CLI로 직접 호출해 bash 파이프로 받으면 Windows에서 cp949로 깨진다 — **스크립트를 쓴다.**
- **자막 실패**(`transcript_ok=false`) 시: 죽지 말고 **제목·설명란만으로** 분류·요약하고, 노트에 `accuracy: low (no transcript)`를 남긴다(특히 짧은 쇼츠가 자막 없는 경우 많음).
- 제목/채널이 비면(oembed 실패) 스크립트가 yt-dlp 값으로 채워준다. 그래도 비면 사용자에게 URL 확인 요청.

### 2. 내용 파악 & 분류

자막(없으면 제목+설명란)을 읽고 **하나의 카테고리**로 분류한다.

| 카테고리 | 폴더(볼트 루트 기준 상대경로) | 무엇인가 |
|---|---|---|
| `ai` | `15. Training/15.03. AI/Youtube` | AI·LLM·Claude/Codex·MCP·플러그인·바이브코딩·프롬프트 등 기술/학습 |
| `recipe` | `05. Private/요리` | 직접 만들어 먹는 **레시피**(특정 제품이 핵심이 아닌 조리법) |
| `shopping` | `05. Private/구매list` | **살 수 있는 특정 제품**(식품·잡화·꿀템). 제품이 핵심이면 여기 |
| `lifetip` | `05. Private/생활 팁` | 제품이 아닌 **순수 노하우/꿀팁**(고르는 법, 단축키, 정리법 등) |
| `inbox` | `00. Inbox` | 분류 애매 / 자막 없어 내용 빈약 → 후분류 |

**분류 핵심 규칙(경계 사례):**
- 레시피를 보여주더라도 **핵심이 "이 제품(소스·도구·완제품) 사라"이면 → `shopping`**. (예: "전설의 일본 소스"로 요리 시연 → 소스가 주인공이므로 구매)
- 제품 없이 **순수하게 "이렇게 만들어 먹어라"면 → `recipe`**.
- 제품 없이 **노하우/팁만이면 → `lifetip`**. "꿀템(살 물건)" 소개는 `shopping`.
- AI/기술/Claude·LLM 관련이면 → `ai`.
- 정말 모르겠거나 자막 없고 제목·설명도 빈약하면 → `inbox`(보류 사유 명시).

### 3. (shopping 전용) 제품 정보 보강

구매로 분류되면 제품명·가격·구매처를 최대한 채운다:
1. **설명란(description)** 먼저 — 제품명·구매 링크·쿠폰·가격이 있으면 사용(해시태그는 `tags`로 활용).
2. 설명란에 없으면 **웹 검색**(WebSearch)으로 실제 상품명·가격·구매처 후보를 찾는다.
3. 웹에서 찾은 건 노트에 **"웹 검색 추정 / 구매 전 확인 권장"**으로 명시(확정 정보와 구분).
4. 그래도 못 찾으면 `(영상 미언급 — 검색 필요)`로 솔직히 둔다.

### 4. 노트 작성 — 카테고리 템플릿 준수

`<skillDir>/references/templates.md`를 읽어 **해당 카테고리 양식 그대로** 작성한다(볼트 정본 우선).

**파일명 규칙** — `YYMMDD_{안전한 제목}.md` (볼트 관행):
- `ai`만 **업로드일** 기준(`upload_date`의 YYMMDD). 나머지는 **작성일(오늘)** 기준.
- 파일명에서 `/ \ : * ? " < > |`·줄바꿈 제거, 공백은 두거나 `-`로.
- `ai`의 frontmatter `Date:`는 **작성일**(시:분 없이 날짜만), 파일명만 업로드일.

**저장 위치**: `{볼트루트}/{카테고리 폴더}/{파일명}`. 폴더가 없으면 만든다.

### 5. 보고

저장 후 1~2줄로:
```
✅ "{제목}" → {category} 로 분류
   저장: {카테고리 폴더}/{파일명}
```
여러 개면 각 줄로 요약한다. 자막 없어 폴백했으면 그 사실도 한 줄.

## 주의

- **볼트 경로 하드코딩 금지** — 항상 `vault_root.txt`로 루트를 받는다(다른 PC·미니PC 이식 대비).
- 영상에 **없는 내용은 지어내지 않는다** — 미언급은 "(영상 미언급)", 추정은 "추정/웹 검색"으로.
- 분류가 정말 애매하면 사용자에게 한 번 확인하거나 `inbox`로 보낸다(억지 분류 금지).
- 배치(여러 링크): 링크별로 1~5단계 반복. 같은 영상이 두 번 들어오면 중복 저장 피한다.
- 자막 추출은 youtube-transcript-api → (실패 시 폴백은 제목·설명란). yt-dlp는 메타데이터·업로드일·설명란 담당.

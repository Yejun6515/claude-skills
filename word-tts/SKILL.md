# word-tts (단어 설명 음성 생성)

일본어 단어 / IT·AI 용어를 입력받아 **설명 대본을 생성**하고 **ElevenLabs TTS로 MP3**를 만든다.
요약·문서추출이 아니라 **Claude가 설명을 직접 작성**하는 생성형 스킬.

## 두 가지 모드

| 모드 | 입력 | 음성 | 스크립트 |
|------|------|------|---------|
| **A. 일본어 단어** | 일본어 단어/동사 | 한국어(Taehyung) **단일** (일본어도 같은 음성이 multilingual로 읽음) | `tts.py` |
| **B. IT/AI 용어** | 영어 기술 용어 | 한국어(Taehyung) **단일** | `tts.py` |

> ⚠️ 이전엔 모드 A를 한국어+일본어 원어민 듀얼 음성으로 만들었으나, 음성 전환이 어색해 **Taehyung 단일 음성으로 통일**했다(구버전 듀얼 스크립트는 삭제됨).

## 설정 (Configuration)

### API 키
- `ELEVENLABS_API_KEY` 환경변수 사용 (이미 등록됨). 미설정 시: `setx ELEVENLABS_API_KEY "sk_..."`
- 플랜: **Starter 이상** (라이브러리 음성 API 사용에 필요). 무료 플랜은 영어 premade 음성만 가능.

### 기본 음성 (확정)
| 용도 | 음성 | Voice ID |
|------|------|----------|
| 모든 해설·단어·예문 (한국어+일본어) | **Taehyung** (남성, 또렷·친근) | `m3gJBS8OofDJfycyA2Ip` |

> 단일 음성. multilingual 모델이라 같은 Taehyung 음성이 한국어·일본어를 모두 읽는다.
> 음성 변경 요청 시 `--voice`로 덮어쓴다.
> (구버전 듀얼용 일본어 음성 Sawaro `EbuvaInXUGWtpYRUnKLQ`는 현재 미사용.)
> 다른 음성 탐색: `client.voices.get_shared(language='ko'/'ja', gender=..., search=...)`

### 모델 / 포맷
- 모델: `eleven_multilingual_v2` (한·일·영 혼합)
- 포맷: `mp3_44100_128` (Starter 호환. Creator+ 면 192 가능)

> ffmpeg는 구버전 듀얼 합치기에만 필요했고, 단일 음성으로 바뀐 지금은 **불필요**하다.

## 모드 A: 일본어 단어 (단일 음성)

### 워크플로우

1. **단어 확인** — 읽기가 여러 개면(예: 潜る = くぐる/もぐる) 어느 읽기인지 확인하거나 대본에서 구분해 설명.

2. **한자 대본(.txt) 작성** — 눈으로 보는 참고용. 한자 그대로 표기.
   - 저장: `{출력폴더}\{한자표기}_대본.txt`
   - 구성: [단어] [뜻] [한자 분석] [뉘앙스] [예문]

3. **TTS 대본(.txt) 작성** — `tts.py` 입력용. 아래 규칙 엄수.
   - 저장: `{출력폴더}\_{한자표기}_대본tts.txt`

4. **생성** (단일 음성, Taehyung):
   ```
   python C:\Users\yejun\.claude\skills\word-tts\scripts\tts.py "<대본tts.txt>" -o "<출력폴더>\한자(히라가나).mp3"
   ```
   (실행 시 `$env:ELEVENLABS_API_KEY` 설정 — 보통 이미 영구 등록되어 있음)

5. **탐색기로 열어** 사용자에게 안내.

### 대본 작성 규칙 (★ 핵심)

단일 음성(Taehyung, multilingual 모델)이 한국어와 일본어를 모두 읽는다. 한국어 문맥 속 한자는 오독되므로:

1. **일본어 단어/예문은 히라가나로 인라인 작성** (`<ja>` 태그 더 이상 사용 안 함).
   - ✅ `くぐる`, `とりいをくぐる。`
   - ❌ `潜る` (한자는 읽기 오류/중의성 위험)
   - 읽기가 중의적인 한자는 히라가나로 써야 의도한 읽기로 발음됨.
   - **언어 전환 지점 앞에 `<break time="0.3s" />`** 를 넣으면 모델이 일본어로 더 잘 전환한다.

2. **한자 분석은 한국어 음으로만.** 대본에 한자 글자(潜 등)를 넣지 않는다.
   - ✅ "이 단어의 한자는 '잠길 잠' 자예요. 삼수변에…"
   - ❌ "潜는 '잠길 잠'…" (음성이 潜을 오독)

3. **한국어 설명 구간에는 한자/가나를 절대 넣지 않는다** (가나를 넣으면 일본어로 읽힘). 한자는 한국어 음으로 풀어 쓴다.

4. **숫자는 한글로** (첫째·둘째, 한 번, 세 가지 등). `references/number_reading.md` 참조.

5. **복합동사 처리 (★ 신규).** 동사 두 개가 결합된 복합동사(예: とりくむ = とる + くむ, ひきうける = ひく + うける)는:
   - 먼저 **앞 동사**의 뜻·한자를 설명하고, 다음 **뒤 동사**의 뜻·한자를 설명한다.
   - 그다음 두 동사가 합쳐져 만들어지는 **복합동사 전체의 뜻·뉘앙스**를 설명한다 (어떻게 결합돼 그런 의미가 되는지 흐름을 짚어줌).
   - 단일 동사면 이 단계 없이 바로 한자 분석으로 진행.

6. 구성: 뜻 → (복합동사면) 앞 동사 분석 → 뒤 동사 분석 → 결합 설명 / (단일동사면) 한자 분석 → (읽기 주의) → 뉘앙스 → **예문 3개**. 끝맺음 "오늘 단어, … 였습니다."

> 한국어 구간 내부 호흡, 언어 전환 지점에 `<break time="0.3s" />` 사용 (스크립트가 그대로 보존). 최대 3초.

### 대본 예시 1 — 단일 동사 (`潜る`)

```
오늘 배울 단어는 くぐる 입니다. 한자로는 '잠길 잠' 자를 쓰는 동사예요. <break time="0.3s" />
먼저 뜻부터 볼게요. 무언가의 아래나 사이를 빠져나가듯 지나가는 것을 말합니다. …
한자를 볼게요. 이 단어의 한자는 '잠길 잠' 자예요. 삼수변에 물속으로 잠긴다는 의미가 담겨…
읽기 주의. 같은 한자라도 <break time="0.3s" /> もぐる 라고 읽으면 잠수한다는 다른 뜻이 됩니다. …
예문 세 개로 마무리할게요.
첫째. <break time="0.3s" /> とりいをくぐる。 도리이 아래를 지나가다.
둘째. <break time="0.3s" /> もんをくぐって なかにはいる。 문을 지나 안으로 들어가다.
셋째. <break time="0.3s" /> きけんを くぐりぬける。 위기를 빠져나가다.
오늘 단어, <break time="0.3s" /> くぐる 였습니다.
```

### 대본 예시 2 — 복합동사 (`取り組む` = とりくむ)

```
오늘 배울 단어는 <break time="0.3s" /> とりくむ 입니다. 동사 두 개가 합쳐진 복합동사예요.
먼저 앞 동사부터 볼게요. <break time="0.3s" /> とる 는 '잡다, 쥐다'라는 뜻이에요. 한자는 '취할 취' 자입니다.
다음 뒤 동사예요. <break time="0.3s" /> くむ 는 '짜다, 맞붙다, 끼다'라는 뜻이에요. 한자는 '짤 조' 자예요.
이 둘이 합쳐지면, 무언가를 손에 단단히 쥐고 정면으로 맞붙는다는 느낌에서 '몰두하다, 임하다, 씨름하다'라는 뜻이 됩니다.
예문 세 개로 마무리할게요.
첫째. <break time="0.3s" /> もんだいに とりくむ。 문제에 몰두하다.
둘째. <break time="0.3s" /> しんけんに しごとに とりくむ。 진지하게 일에 임하다.
셋째. <break time="0.3s" /> あたらしい かだいに とりくむ。 새로운 과제에 씨름하다.
오늘 단어, <break time="0.3s" /> とりくむ 였습니다.
```

## 모드 B: IT/AI 용어 (단일 한국어 음성)

AI 공부 중 모르는 용어를 쉽게 설명하는 한국어 음성.

### 워크플로우
1. **대본 작성** — 한국어 구어체. **영어 약어는 한글 음가로** 쓴다 (RAG→래그, AI→에이아이, LLM→엘엘엠, ChatGPT→챗지피티). 영어 풀네임도 한글 음가로 (Retrieval Augmented Generation→리트리벌 오그멘티드 제너레이션).
   - 한국어 음성이 영어를 어색하게 읽으므로 **로마자 알파벳을 대본에 직접 쓰지 않는다.**
2. **생성** (단일 음성, Taehyung):
   ```
   python C:\Users\yejun\.claude\skills\word-tts\scripts\tts.py "<대본.txt>" -o "<IT-AI폴더>\용어(한글발음).mp3"
   ```
3. 구성: 용어 소개(한글발음 + 영어 풀네임 한글음가 + 우리말 뜻) → 쉬운 설명 → 동작/비유 → 어디에 쓰는지.

### 대본 예시 (`RAG`)
```
오늘 알아볼 용어는 래그입니다. 영어로는 리트리벌 오그멘티드 제너레이션, 우리말로는 검색 증강 생성이에요.
쉽게 설명해 볼게요. 챗지피티 같은 거대 언어 모델은 똑똑하지만, 학습한 시점까지의 지식만 알고 있어요. …
비유하자면, 닫힌 책 시험이 아니라 오픈북 시험을 보게 해 주는 거예요. …
오늘 용어, 래그였습니다.
```

## 배치 (여러 단어 한꺼번에)

사용자가 단어 목록을 주면, **단어마다** 모드 A(또는 B) 워크플로우를 반복한다.
- 각 단어의 대본을 순서대로 작성 → 각각 생성 → 파일명 규칙대로 저장.
- 진행 상황을 `[1/N] 単語… ✅` 형태로 보고.
- 배치 MP3 파일명을 **순번 접두**로 요청할 수 있다(스마트 글라스 학습용): `순번. 단어.mp3` 예) `1. 釣れる.mp3`.

### 스마트 글라스용 단어 정리 txt (배치 동반 산출물)
사용자가 스마트 글라스에서 볼 단어 목록을 요청하면, MP3와 함께 **간단 정리 txt**도 만든다.
- **간단히**: 단어(읽기) + 뜻 + **각 한자의 한국 한자음(훈+음)**. 예문·대본 전체 문장은 **넣지 않는다**.
- 한자별 한국음은 ` / `로 구분. 가나·외래어 단어는 `(한자 없음 · 가나)` / `(한자 없음 · 외래어)`로 표기.
- **번호 사이는 한 줄 띄운다.**
- 예시:
  ```
  1. 釣れる (つれる)
  낚이다, 잡히다 (자동사)
  釣 낚을 조

  2. 天井クレーン (てんじょうクレーン)
  천장 크레인
  天 하늘 천 / 井 우물 정
  ```

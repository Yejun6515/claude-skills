# 見積計算書(詳細) 읽기 — 품목별 Eng/Manu 추출 (breakdown용)

고객 제출 breakdown(예: POSCO K3CX_ME)을 만들 때, 각 라인의 **Eng(설계비)·Manu(제작비)** 는 `見積計算書(詳細)` 시트에서 추출한다. (Rev7b 실데이터로 검증된 절차)

> ⚠️ **컬럼 위치는 견적서마다 다를 수 있다.** Rev가 올라가며 열이 추가됨. **항상 행 10~11 헤더의 텍스트로 컬럼을 찾을 것**(고정 letter 신뢰 금지). 좌측 本案 + 우측 代案/HSG 두 블록이 나란한 경우 → **本案(좌측) 블록** 사용.

## 핵심 추출 공식 (검증됨)
- **Eng = 設計費(社内) + 設計費(社外)**
- **Manu = MAIN ITEM 原価計 − Eng**
- **품목코드(A열, 예 `1-11 2)`)로 group**, Eng·Manu 합산 → 피벗

헤더(행11) 텍스트 ↔ 대략 위치(K3CX Rev7b 기준, 파일마다 시프트):
| 헤더 텍스트 | 대략 열 |
|---|---|
| `設計費(社内)` | X (24) |
| `設計費(社外)` | Y (25) |
| `MAIN ITEM 原価計` | AC (29) |
| 品目コード(`Spec.`/`Item No.`) | A (1) |

## 스코프 한정 (중요)
- **기계(PTJ) 품목 행만** 합산. K3CX Rev7b에선 **행 12~146**.
- 그 아래는 제외: **電機(코드 137/138)**, **합계·중복 테이블**(거대값 행), `総括費用のみ` 별도 테이블(예 행 164~173 = 별도 합산).
- A열 토큰 제외: `無し`, `不要`, `-`, `*`, `デフォルト/必須`, `137`, `138`.
- blank 코드 행 = 총괄(冷プロ総括 등) → 이름(C열)으로 매칭.

## 검증 체크
- **설계공수(Eng)는 스코프 불변이면 Rev 간 거의 동일**해야 한다. Eng 급증 = 신규 설계(스코프 변경), Eng 불변·Manu만 상승 = **제작원가 재산정**(스코프 변경 아님).
- 합산 Σ原価計을 `見積まとめ`의 製造原価 / 製造原価計과 교차검증.
- 피벗을 **블록(블록1/Rev2 등)에 흘려 기존 값 재현되면(FAIL=0)** 라인↔코드 맵 신뢰.

## Excel/COM 함정
- 코드 `1-5`·`1-6`·`1-10` 등은 **Excel이 날짜로 자동변환**("01月05日"). 라벨 열 **NumberFormat="@"(텍스트) 먼저 지정** 후 기입.
- 코드 정규화 시 전각 공백(`　`)·전각 괄호(`（）`) → 반각 통일.
- 분할 코드: 한 코드가 여러 라인에 쪼개지면(예 `1-11 9)`=기어박스+스핀들) **부품행(B열 부품번호) 단위로 분할**. 합쳐진 코드(예 `1-11 4)/1-11 5)`)는 피벗 라벨 그대로.

## PowerShell COM 추출 스니펫 (참고)
```powershell
function Norm($s){ ([string]$s) -replace '[\r\n]','/' -replace '　',' ' -replace '\s+','' -replace '（','(' -replace '）',')' }
$excl=@('無し','不要','-','*','137','138','デフォルト/必須')
# rows 12..146, code=col1, 設計社内=col24, 設計社外=col25, 原価計=col29
foreach($r in 12..146){
  $code=Norm $sh.Cells.Item($r,1).Text; if($excl -contains $code){continue}
  $ac=[double]$sh.Cells.Item($r,29).Value2; $x=[double]$sh.Cells.Item($r,24).Value2; $y=[double]$sh.Cells.Item($r,25).Value2
  $engi=$x+$y; $manu=$ac-$engi   # group by $code, sum
}
```
